    \block{1. Extraction Strategy}{
        We built a pipeline that automates the creation of image pairs $(I_A, I_B)$ from a youtube video called \textit{10 Hours Fantastic Views of Nature 4K with Relaxation Music}.
        The video is a compilation of drone views of landscapes which moves slowly forward. We extracted different pairs of images separated by a gap of 4 frames every 0.05 seconds to teach the model how to reconstruct successive images.
        We limited this number of pairs to 10,000 for the first, small, dataset and 100,000 for the bigger one.\\
        We did not use the original format of the dataset. The video stream was reduced to 24 FPS and the extracted images were cropped to squares of 512 pixels and centered.
    }

    \block{2. Intelligent Data Filtering}{
        The Youtube video is a succession of landscape clips. To remove the pairs of pictures belonging to 2 different video clips, we used a distance metric.
        We defined 2 different metrics:
        \begin{itemize}
            \item \textbf{Hist Distance}: Bhattacharyya ($>0.35$).
            \item \textbf{Thumbnail MAE}: Structural difference. Compares two lower resolution images in grayscale ($>0.25$).
        \end{itemize}
        These 2 metrics are necessary because MSE is insensitive to semantic "cuts".
        Combined together, they ensure the model only learns real movement and not montage jumps.
    }

    \block{4. Motion Flow Loss (Warping)}{
        Since we don't have "ground truth" motion vectors, we use a \textbf{self-supervised reconstruction loss}:
        \begin{itemize}
            \item The model predicts a flow $(dx, dy)$ from image $I_A$.
            \item We \textbf{warp} $I_A$ using this flow to reconstruct $I_B$.
            \item The loss is the Mean Squared Error (MSE) between the predicted $I_{B'}$ and the real $I_B$:
            \[ \mathcal{L} = \| \text{Warp}(I_A, \text{Flow}) - I_B \|^2 \]
        \end{itemize}
        \begin{center}
            \includegraphics[width=0.85\linewidth]{motion_flow_example.png}
        \end{center}
    }

    \block{6. SVD Training Strategy}{
    We explored two fine-tuning approaches on the same dataset of 2374 motion-filtered clips:

    \vspace{0.5em}
    \textbf{Approach A — LoRA on temporal attention layers:}
    \begin{itemize}
        \item Only $\sim$3.3M parameters trained (0.2\% of UNet)
        \item Rank 16, $lr=10^{-4}$, 7 epochs, batch size 2
        \item Naive MSE on EDM noise prediction
    \end{itemize}

    \vspace{0.5em}
    \textbf{Approach B — Full fine-tuning via SVD\_Xtend:}
    \begin{itemize}
        \item All UNet parameters trained
        \item $lr=10^{-5}$, 5000 steps, \texttt{noise\_aug\_strength=0.02}
        \item Correct EDM loss with CFG dropout
    \end{itemize}

    \block{7. SVD Results \& Perspectives}{
    \textbf{Motion Flow results:}
    \begin{itemize}
        \item \textbf{Strengths:} Easy to use, fast and close from the initial image.
        \item \textbf{Limitation:} Cannot handle small details. Treats the image as a whole and do not create any pixel (only displacements).
        \item \textbf{Impact:} Interesting for tiny animations but useless in practice.
    \end{itemize}
